## Spacing Scale

**Declared scale (multiples of 4 only):**

| Token | Value | Usage in Phase 2 |

| sm | **8px** | Compact stack gaps, gap between set chips in cinta |

| 2xl | **48px** | Vertical breathing around hand-off overlay content |
| 3xl | **64px** | Page-level top padding on resumen card / safe-area bottom for strip |

**Exceptions (explicit, locked):**
- **Touch targets:** Primary CTAs (`‚úì Hecho`, steppers `+`/`-`, "Saltar", "Empezar ya", "Iniciar nueva sesi√≥n") have a hit area of **`min-height: 56px; min-width: 56px`** per **D-21**. - **Legacy Phase 1 components** (`.set-row`, `.radio-card`, `.input`) currently use `12px` and `10px` paddings. They are **grandfathered**; no Phase 2 work changes them. New Phase 2 components (FocusCard, ExerciseStrip, RestStrip, RestPanel, HandoffOverlay, SummaryScreen, Toast, PauseDialog) use only the scale above.
- **Set chip in cinta superior:** chip itself is `40√ó32` (height √ó min-width). The hit area extends to `48√ó48` via padding/transparent inset. Ensures comfortable tap without enlarging visual footprint.

## Typography

**Declared roles (4 sizes, 2 weights):**

| Role | Size | Weight | Line Height | Usage in Phase 2 |
|------|------|--------|-------------|------------------|
| Body | **16px** | **400** (regular) | **1.5** | Hint text, microcopy under steppers, RIR chip labels, exercise name in chip strip |
| Label | **13px** | **700** (bold) | **1.2** | Uppercase labels (D-02 indicators: "EJERCICIO 2 DE 4", "GUARDADO", "DESCANSO"), strip status copy, RIR chip number |
| Heading | **22px** | **700** (bold) | **1.2** | Current exercise name in focus-card header, summary card title, hand-off "Siguiente: ‚Ä¶", pause-prompt question |
| Display | **`var(--display-set)` = clamp(48px, 14vw, 72px)** | **700** (bold) | **1.0** | XL set numbers in focus card (peso, reps), countdown mm:ss in expanded RestPanel |

**Weight policy:** **exactly 2 weights** ‚Äî `400` and `700`. The legacy `font-weight: 650` used in Phase 1 (`.label`, `.btn`, `.set-left strong`) is **grandfathered** in existing classes; new Phase 2 components only use 400/700.

**Font-feature settings:** `font-variant-numeric: tabular-nums` is required on:
- The countdown `mm:ss` text in the RestStrip and RestPanel
- The XL weight/reps display in the FocusCard
- The `‚úì 92s ‚Üë+2s` per-set chip metadata
This prevents glyph-width jitter as digits change every second / on stepper press.

**Letter-spacing:**
- Display role: `-0.02em` (tight, dramatic)
- Heading role: `-0.3px` (matches existing `.section-title`)
- Label role: `0.2px` UPPERCASE (matches existing `.label`)
- Body role: `0` (default)

**Reading-distance check:** D-21 requires legibility @ ~50 cm. The `clamp(48px, 14vw, 72px)` Display rule yields ‚â•48px at 360px viewport (worst-case mobile) ‚Äî confirmed legible at the required distance. Tested mentally against typical phone screen at arm's length.

## Interaction & Motion

**Locked principles (D-21 + Claude's Discretion):**

| Element | Behavior |
|---------|----------|
| **Tap targets** | Primary: ‚â• 56√ó56 px (D-21). Secondary: ‚â• 44√ó44 px. Chips with extended hit area: visual 40√ó32 + transparent inset to 48√ó48. |
| **Long-press steppers** | Initial 200 ms threshold ‚Üí tick every 80 ms thereafter (D-05 "long-press para acelerar"). Visual: subtle scale 1.05 while active. |
| **Toast entry/exit** | Entry: 180 ms slide-up + fade (cubic-bezier(0.2, 0.8, 0.2, 1)). Exit: 140 ms slide-down + fade. |
| **Hand-off entry/exit** | Entry: 220 ms fade + scale 0.96 ‚Üí 1. Persists ~3000 ms. Exit when accepted or `visibleUntil` reached: 140 ms fade. |
| **Active set chip transition** | 160 ms ease for border-color + scale (1 ‚Üí 1.04) when becoming active. |
| **Rest end visual flash** | `--accent-flash` background pulses twice over 1500 ms (keyframes 0% ‚Üí 50% ‚Üí 100%; opacity envelope). Honors `prefers-reduced-motion: reduce` ‚Üí solid color hold for 1500 ms, no pulse. |
| **Stepper press feedback** | 80 ms transform: scale(0.96) on active, releases instantly. |
| **Reduced motion** | All transitions ‚â§ 100 ms or removed under `@media (prefers-reduced-motion: reduce)`. The flash becomes a static color, not a pulse. The hand-off does NOT auto-advance with motion; the countdown copy still updates ("Empieza en 2‚Ä¶"). |

**Sound and haptics (D-11, RESEARCH ¬ßAudio Cues):**
- Beep: synthesized 880 Hz sine, 180 ms, 0.4 gain, soft attack/release. Optional double beep (880 Hz + 660 Hz, 100 ms gap) for emphasis ‚Äî Claude's Discretion.
- Vibration: pattern `[150, 100, 150]` at rest end. iOS silent fallback.
- Both gated by `Preferences.restAlertSound` / `restAlertVibration` (default ON, D-24).
